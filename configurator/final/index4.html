Skills Used in Creating this Project

My project was unique in that I was working with already created designs to implement a concept. It is not unusual to draw inspiration from other sources but because my fonts and styles came from an already established style guide, I wanted to list out the skills I used for this project:

  • Working off of an established style guide
  • Matching fonts, line weights and color schemes in my work so it fits seamlessly into the project.
  • Identifying a specific user need and finding a way to address it within the constraints presented by a client/stakeholder
  • When talking to my boss he noted the infeasibility of creating a whole new website for a ‘Neighborhood Report’ configurator. While creating a survey type configurator may have addressed the prompt for the class more effectively/straightforwardly, I wanted to find a solution that may actually be implementable while still being a configurator.
  • Recreating styles in sketch
  • I did not screenshot anything besides the front page map, everything else was recreated or created from scratch in sketch
  • Communicating a concept for a user without much prior explanation
  • I purposely decided not to include a long explanation of how the pinning system works on the website. Because we are accustomed to adding things to carts, ‘starring’ or ‘pinning’ things of interest, I wanted to draw on that user tendency in a different context. While a little experimentation I think my concept would be easy to understand for most user types.
  • Pitching a project/feature expansion to an organization by laying out my process, reasoning behind design decisions, and creating a clickable mockup to showcase my vision
  • I plan on pitching this project to the company next week because I think it addresses an important need. We’ll see how it goes but I think it’s invaluable practice.
